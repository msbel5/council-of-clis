# Project status (template)

Edit this block to give every CLI the same project context. It's prepended to every prompt
you send via Council. Keep it concise — every CLI re-reads it on every prompt.

The local file `status.md` is gitignored. This example file is committed.

## Who you're talking to

- Your name and role (one line)
- Communication preference (e.g. "Turkish-first, English when I open English")
- Tone (e.g. "no hedging, cite file paths, treat me like a senior engineer")

## Current project

- Project name and one-paragraph description
- Repo (if applicable)
- Tech stack you're working in
- Active branch and what it's doing

## Goals this week

- 2-3 concrete deliverables
- Acceptance criteria for each

## Constraints

- Cost: which API budgets are tight, which are loose
- Quality: which lint/test bars must hold
- Phase fences (anything you're NOT supposed to do yet)

## Recent decisions

- 3-5 bullets summarizing decisions worth keeping in context

## What "done" looks like

- Concrete signal (e.g. "PR opens with X tests passing, Y validation gate green")

## Tone for replies

- Reply format you expect (markdown blocks? code-only? file-path-cited?)
- Length cap (e.g. "max 500 words unless I ask for full code")
